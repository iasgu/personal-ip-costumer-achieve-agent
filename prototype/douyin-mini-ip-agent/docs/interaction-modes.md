# 评论互动运营双模式

## 模式定义

### 智能辅助模式

当前默认模式。小程序拉取评论后，为单条评论生成候选回复，运营人员可以选择候选、手动编辑，再点击确认回复。

适用阶段：

- 真实抖音 API 刚接入时，需要人工看结果。
- 账号口吻还在调整时，需要运营人员改文案。
- 涉及争议、投资、承诺类评论时，需要人工兜底。

### 全自动模式

用户点击开始后，小程序按评论优先级逐条执行：

1. 读取待回复评论。
2. 调用后端生成候选回复。
3. 取第一条候选回复作为自动回复内容。
4. 调用后端回复接口。
5. 在页面日志里记录生成、提交、成功或失败原因。

后端仍然保留风险拦截。如果回复内容触发高风险规则，接口会拒绝提交，小程序会把失败原因展示出来。

## 当前接口依赖

- `GET /api/comments?itemId=...`：拉取评论列表。
- `POST /api/comments/:id/suggestions`：生成单条评论候选回复。
- `POST /api/comments/:id/reply`：提交回复。
- `GET /api/strategy/current`：读取当前回复策略。

## 接真实抖音 API 时的处理

前端不需要大改。后端把 `COMMENT_ADAPTER` 从 `mock` 切到真实抖音适配器，并配置开放平台 access token、权限和 itemId 映射即可。

建议上线顺序：

1. 先开放智能辅助模式，人工确认后发。
2. 小流量测试全自动模式，只处理高优先级、低风险评论。
3. 稳定后再扩大到全部待回复评论。
