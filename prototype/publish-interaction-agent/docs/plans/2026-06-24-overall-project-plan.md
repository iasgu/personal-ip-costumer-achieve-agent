# C端个人IP智能体 全盘推进计划

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 在不混淆已有项目的前提下，把「爆款链接生成真人口播视频」和「一键发布/互动评论」拆成可演示、可拼接、可商业化验证的模块体系，并在 2026-06-25 上午准备好给西安测试。

**Architecture:** 当前采用模块化推进：`ip-agent` 负责内容生成链路，`publish-interaction-agent` 负责发布草稿与评论互动，模型/平台能力通过 adapter 隔离。短期先用 mock/local 保证演示闭环，中期接抖音 OpenAPI 和即梦 API，长期抽象为可配置工作流。

**Tech Stack:** Node.js 本地 HTTP 服务、原生 HTML/CSS/JS、JSON 本地存储、DashScope/Qwen/Wan、火山引擎即梦、抖音开放平台 OpenAPI、后续可接数据库和对象存储。

## 1. 项目总目标

做一个面向 C 端个人 IP 创作者的智能体工具：

```text
给一个爆款视频链接 + 上传头像/声音
-> 自动分析爆款结构
-> 改写成适合本人定位的口播脚本
-> 生成真人口播视频
-> 生成发布草稿
-> 发布到平台
-> 拉取评论并生成拟人回复
```

核心价值：

- 降低普通人做个人 IP 的内容生产门槛。
- 支撑账号日更、多账号测试、快速迭代。
- 把“内容生成”延伸到“发布运营”和“评论互动”，形成闭环。

## 3. 总体架构

### 3.1 模块边界

```text
ip-agent
  负责：爆款分析、脚本改写、音频生成、视频生成
  输出：视频文件、标题、文案、话题、内容包

publish-interaction-agent
  负责：发布草稿、发布状态、评论列表、回复建议、人工确认回复
  输入：视频文件、标题、文案、话题、个人定位、知识库

model/provider adapters
  负责：Wan、即梦、Qwen、后续其他模型切换

platform adapters
  负责：mock、本地测试、抖音 OpenAPI、后续视频号/小红书
```

### 3.2 数据流

```text
用户素材
  -> 头像/音频/抖音链接
  -> ip-agent 生成内容包
  -> publish-interaction-agent 创建发布草稿
  -> 平台 adapter 提交发布
  -> 评论 adapter 拉取评论
  -> 回复建议引擎生成候选回复
  -> 人工确认
  -> 平台 adapter 回复评论
```

### 3.3 为什么先拆开

- 避免今天开发影响昨天已经能跑的生成链路。
- 抖音 OpenAPI 权限未通过时，也能独立演示发布和评论逻辑。
- 后续可通过 API 拼接，不需要一开始就把项目揉成一团。

## 8. 风险清单

### 平台权限风险

抖音真实发布和评论回复必须等开放平台权限。

应对：

- 今天用 mock 演示产品闭环。
- adapter 预留真实接入。

### 模型质量风险

Wan 或即梦生成的人脸、嘴型可能不稳定。

应对：

- 多 provider 对比。
- 保留人工审核。
- 先支持 15 秒短视频。

### 成本风险

1 分钟视频如果拆分生成，成本和耗时会放大。

应对：

- 先测试 15-30 秒。
- 记录单条成本。
- 并行生成。
- 优先用更便宜且稳定的 provider。

### 合规风险

自动发布、自动评论容易触发平台规则。

应对：

- 使用官方 OpenAPI。
- 评论回复保留人工确认。
- 不做违规浏览器自动发布。

### 交付混乱风险

多个原型混在一起会难维护。

应对：

- 当前按目录拆分。
- 通过 API contract 拼接。
- 每个模块单独 README 和计划。
