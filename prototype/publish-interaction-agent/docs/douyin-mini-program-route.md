# 抖音小程序路线调整

更新时间：2026-06-24

## 结论

如果当前只能开发抖音小程序，那么本项目应调整为：

```text
抖音小程序 = 用户前端入口
服务器后端 = 模型调用、视频生成、素材存储、发布/评论 OpenAPI 转接
```

不要再按“网站应用前端直接 OAuth + 发布工具”设计。

## 官方能力边界

### 小程序可做

1. 作为用户入口：
   - 上传头像
   - 录制/上传声音
   - 输入抖音链接
   - 查看生成结果
   - 管理发布草稿
   - 查看评论回复建议

2. 调起授权：
   - 小程序 JS API 中有 `tt.showDouyinOpenAuth`，用于展示抖音权限授权弹窗。
   - 使用前需要在开放平台管理中心给小程序申请对应权限。

3. 视频数据：
   - 官方“视频数据”能力支持小程序、移动应用、网站应用。
   - 可在用户授权后查询用户视频列表和视频基本信息。

4. 视频挂载小程序：
   - 需要小程序质量评价达到 A 级后，申请视频挂载小程序权限。
   - 可用于后续增长闭环，让发布的视频挂载我们的小程序。

### 小程序不应直接做

1. 不在小程序端保存平台密钥、DashScope Key、火山引擎 Key。
2. 不在小程序端直接调模型生成。
3. 不在小程序端保存 access_token / refresh_token。
4. 不绕过官方能力做自动发视频。

## 发布能力变化

官方“发布内容至抖音”能力页显示支持应用类型为：

```text
移动应用、网站应用
```

因此如果只有小程序应用，不能简单照搬网站应用的“直接发布 OpenAPI”方案。

小程序路线更现实的发布策略是：

### 方案 A：先做发布辅助，不承诺自动发布

```text
生成视频
-> 小程序展示成片
-> 提供标题/话题/文案
-> 用户下载或保存视频
-> 用户手动发抖音
-> 后续通过视频数据能力拉取已发布视频数据
```

优点：最快能上线、合规风险低。

缺点：不是完全一键发布。

### 方案 B：小程序内调起抖音发布/拍摄相关能力

需要继续查小程序是否具备对应“拍抖音/发布视频”JS API，并确认是否能传入服务器生成的视频文件。

如果平台只允许用户拍摄或从相册选择，则后端生成视频后仍需用户保存到本地再选择发布。

### 方案 C：补一个网站应用或移动应用

如果后续一定要“服务器一键发布到抖音”，建议额外创建网站应用或移动应用，用它承接内容发布 OpenAPI。

小程序继续作为 C 端入口。

## 评论互动能力

评论管理需要申请互动管理权限：

```text
Scope: video.comment
```

接口包括：

```text
GET  /video/comment/list/        评论列表
GET  /video/comment/reply/list/  评论回复列表
POST /video/comment/reply/       回复视频评论
POST /video/comment/top/         置顶视频评论，仅企业号可用
```

小程序前端只展示评论和回复建议；真实评论列表和回复动作应由服务器后端调用。

## 新产品形态

小程序首页推荐简化为 4 步：

```text
1. 上传头像/真人形象
2. 录制或上传声音
3. 粘贴爆款抖音链接
4. 生成视频
```

生成后进入：

```text
发布草稿页
  - 视频预览
  - 标题
  - 话题
  - 发布文案
  - 保存视频/复制文案/去抖音发布

互动页
  - 选择已发布视频
  - 拉取评论
  - 生成回复建议
  - 人工确认回复
```

## 今天执行调整

原计划里的 `publish-interaction-agent` 不废弃，但定位变成：

```text
小程序后端原型 + 浏览器调试台
```

接下来新增一个小程序前端目录：

```text
prototype/douyin-mini-ip-agent
```

里面只做小程序页面和 API 调用，不放任何密钥。

后端继续复用：

```text
prototype/ip-agent
prototype/publish-interaction-agent
```

## 给用户要做的事

1. 在抖音开放平台确认小程序 AppID。
2. 确认小程序后台可申请的能力列表。
3. 优先申请：
   - 视频数据能力
   - 互动管理 / video.comment
   - 视频挂载小程序能力，若满足评级条件
   - 如有“视频发布/拍抖音”类能力，也一起申请
4. 不要把账号密码、client_secret、access_token 发到文档或前端代码。
