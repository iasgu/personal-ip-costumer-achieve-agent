# 即梦能力矩阵

更新时间：2026-06-24

用途：判断即梦哪些能力适合并入「个人 IP 智能体」工作流。本文先做产品和技术判断，不写真实密钥，不记录账号密码。

## 总体判断

即梦不应该只当成一个视频生成模型看，它更接近一组内容生产 API：

- 素材生成：文生图、图生图、智能参考、局部重绘、扩图、超清。
- 视频生成：文生视频、图生视频、视频 3.0 Pro。
- 真人/数字人：数字人快速模式、OmniHuman、动作模仿。
- 智能成片：小云雀智能生视频 Agent、小云雀营销成片 Agent。

对我们最有价值的是「真人形象口播」和「营销成片 Agent」两条线。

## 能力表

| 能力 | 输入 | 输出 | 适合环节 | 风险 | 优先级 |
|---|---|---|---|---|---|
| 数字人快速模式 | 人像图、音频、脚本 | 口播视频 | 用户头像 + 用户音频生成真人口播 | 嘴型、脸部稳定性、授权成本 | 高 |
| OmniHuman1.5 | 人像/视频、音频等 | 真人驱动视频 | 更高质量真人形象生成 | 调用门槛、耗时、成本未知 | 高 |
| 小云雀营销成片 Agent | 产品信息、脚本、素材 | 营销视频 | 快速生成投放/私域风格成片 | 是否能吃个人定位和知识库，需要实测 | 高 |
| 小云雀智能生视频 Agent | 主题、素材、脚本 | 视频 | 从选题到成片的自动化 | 可控性可能弱于自研工作流 | 中高 |
| 视频生成 3.0 Pro | 文本/图片提示 | 视频片段 | 生成补充镜头、B-roll、转场 | 未必适合真人口播 | 中 |
| 动作模仿 2.0 | 参考动作、人物素材 | 模仿动作视频 | 模仿爆款视频肢体节奏 | 可能不适合严肃 IP 口播 | 中 |
| 图生图智能参考 | 参考图、提示词 | 风格/人物一致图片 | 封面、头像、背景统一 | 人物一致性需验证 | 中 |
| 文生图/图片生成 | 提示词 | 图片 | 封面、海报、背景素材 | 和主流程关系不如视频强 | 低中 |
| 扩图/局部重绘/超清 | 图片 | 修复图片 | 提升头像、封面素材质量 | 是辅助能力，不是核心闭环 | 低 |

## 和现有工作流的对齐方式

### 1. 爆款视频分析

继续由我们当前链路负责：

```text
抖音链接 -> 录屏抽帧 + ASR -> 多模态理解 -> 爆款结构/钩子/转化点
```

即梦不替代这一层，因为我们需要可解释结构和可控改写。

### 2. 个人定位和知识库

由我们做 Prompt 和约束层：

```text
个人定位 + 禁用词 + 行业知识库 + 案例库 -> 即梦输入提示词/脚本/素材约束
```

即梦负责生成，不负责业务判断。

### 3. 真人口播生成

优先对比：

```text
Wan2.7-i2v vs 即梦数字人快速模式 vs OmniHuman1.5
```

评估指标：

- 嘴型同步
- 人脸稳定
- 是否像本人
- 单条耗时
- 单条成本
- 最长可生成时长
- 是否支持并行
- 是否适合批量日更

### 4. 营销成片

重点看小云雀营销成片 Agent：

```text
爆款结构 + 改写脚本 + 用户素材 + 个人定位 -> 营销成片
```

如果它可控性好，可以用于低成本 MVP；如果可控性差，则只作为参考工具。

## 今日优先测试顺序

1. 数字人快速模式：最贴近“头像 + 音频 -> 真人口播视频”。
2. OmniHuman1.5：判断能否作为高质量真人形象方案。
3. 小云雀营销成片 Agent：判断能否替代部分剪辑和包装。
4. 视频生成 3.0 Pro：用于 B-roll 或补镜头，不作为主链路优先项。

## 暂定结论

即梦值得接入成 adapter，但不建议今天直接替换主流程。

更稳妥的路线：

```text
先保留我们自己的分析/改写/知识库层
再把视频生成层抽象为 adapter
Wan、即梦、后续其他模型并列测试
最终按成本/质量/耗时选择默认方案
```
